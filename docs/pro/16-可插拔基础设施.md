## 2. 可插拔配置矩阵

### 2.1 各服务 application.yml 默认值

| 配置项 | 环境变量 | 默认值 | Docker 接入方式 |
|--------|---------|--------|----------------|
| Redis 主机 | `REDIS_HOST` | `localhost` | Docker 启动后自动连接 |
| Redis 密码 | `REDIS_PASSWORD` | 空 | `docker compose` 自动注入 |
| Nacos 注册 | `NACOS_ENABLED` | `false` | 设置为 `true` + 启动 Nacos |
| Nacos 服务器 | `NACOS_SERVER` | `nacos:8848` | Docker 网络 DNS |
| 配置源 | `CONFIG_SOURCE` | `properties` | 切换为 `nacos` |
| MySQL | `DB_HOST` | H2 内存 | 切换为 `mysql:3306` |
| Kafka | — | 未接入 | Phase 3 补充 |


## 3. 代码级容错实现

### 3.1 Redis 健康检查容错（已实现）

**现象**：Redis 未启动时，`/actuator/health` 返回 `DOWN`，日志中每 10 秒打印一次连接失败警告。

**代码路径**：`spring-boot-starter-actuator` 自动配置 → `RedisReactiveHealthIndicator`

**处理方式**：服务不崩溃。限流降级为 Resilience4j 内存模式（`spring-cloud-starter-circuitbreaker-reactor-resilience4j` 无需 Redis）。

**接入 Docker Redis**：
```bash
# 方式一：环境变量
export REDIS_HOST=localhost
export REDIS_PASSWORD=manpou123

# 方式二：启动 docker compose
docker compose -f docker/compose.yaml up -d redis
```

### 3.2 Nacos 容错（已实现）

```yaml
# application.yml 中已配置
spring.cloud.nacos.config.enabled: false    # 默认禁用
spring.cloud.nacos.discovery.enabled: false # 默认禁用
```

**接入 Nacos**：
```bash
# 1. 启动 Nacos
docker compose -f docker/compose.yaml up -d nacos

# 2. 设置环境变量
export NACOS_ENABLED=true
export NACOS_SERVER=nacos:8848

# 3. 重启服务
mvn spring-boot:run -DskipTests
```

### 3.3 Kafka 降级（待实现 Phase 3）

```
Kafka Broker 可用：
  发注单审批通过 → Kafka Topic → 各服务消费者

Kafka Broker 不可用：
  发注单审批通过 → Outbox 表（已有 V2 迁移）→ 轮询补偿 Job
```

Outbox 表已在 `V2__outbox_table.sql` 中定义，补偿 Job 待 Phase 3 实现。

### 3.4 限流降级（已实现）

```
Redis 可用：Resilience4j + Redis 计数器（原子限流）
Redis 不可用：Resilience4j 内存模式（令牌桶）
```

配置在 `resilience4j.ratelimiter.default`，无需修改代码。

## 4. 快速启用 Docker 基础设施

### 4.1 一键启动（推荐）

```bash
# 在项目根目录
docker compose -f docker/compose.yaml up -d

# 验证所有容器
docker compose -f docker/compose.yaml ps
```

### 4.2 按需启动（仅核心组件）

```bash
# 仅 MySQL（用于 Phase 2 采购单开发）
docker compose -f docker/compose.yaml up -d mysql

# 仅 Redis（用于限流测试）
docker compose -f docker/compose.yaml up -d redis

# MySQL + Redis + Kafka（全量 Phase 2-3）
docker compose -f docker/compose.yaml up -d mysql redis kafka
```

### 4.3 查看各容器端口

```bash

# nacos             Up        0.0.0.0:8848->8848/tcp
# prometheus        Up        0.0.0.0:9090->9090/tcp
# grafana           Up        0.0.0.0:3001->3001/tcp
```

## 6. 单镜像打包（Demo 模式）

**用途**：本地开发/Demo 演示，一条命令启动全部后端。

```
ManpouChinaSystem-allinone.jar
├─ user-service        (18081)
├─ product-service     (18082)
├─ procurement-service (18083)
├─ warehouse-service   (18084)
├─ customs-service     (18085)
├─ logistics-service   (18086)
├─ finance-service     (18087)
├─ notification-service(18088)
└─ api-gateway         (18080)
```

**打包方式**（Phase 4）：
```bash
# 构建
mvn clean package -DskipTests

# 运行（使用 Docker MySQL/Redis/Kafka）
docker compose -f docker/compose.yaml up -d mysql redis kafka
java -jar ManpouChinaSystem-allinone.jar
```

> **注意**：单镜像不适合生产（扩缩容 / 日志隔离 / 故障隔离均失真）。
> 仅用于本地演示和 CI 环境。
