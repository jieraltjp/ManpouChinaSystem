## 1. 需求背景

面向中日贸易场景的采购发注全链路管理系统，覆盖从「任务启动」到「流程闭环」的全流程数字化。

**核心价值链**：
- 厂家管理 + 商品信息录入
- 多运输模式分流（自有仓 / 拼柜 / 直装）
- 拼柜智能合箱策略
- 货柜管理与财务结算
- 全链路状态追踪

## 3. 业务状态机

```
发注单状态流转：

PENDING(待执行)
    │ ──→ SUSPENDED(已挂起) ──→ PENDING
    │
    └─→ IN_PROGRESS(执行中)
              │
              ├─→ QC_PENDING(待验收)
              │         │
              │         ├─→ REJECTED(返工) ──→ PENDING
              │         │
              │         └─→ QC_PASSED(验收通过)
              │
              └─→ SHIPPING(已发货)
                        │
                        └─→ CLOSED(流程闭环)
```

---

## 4. 运输模式

| 模式 | 触发条件 | 路径 |
|------|----------|------|
| 自有仓库 | 货量充足 | → 入自有仓 → 二次验收 → 集货 |
| 虚拟拼柜 | 货量不足 | → 拼柜池等待 → 凑柜 → 巡回提货 |
| 厂家直装 | 紧急/大批量 | → 厂家地址直接装柜 + 报关并行 |

---

## 5. 拼柜策略

**触发条件**：目的地相同 + 凑够一整柜（以 CBM 或箱数为单位）

**自动汇总字段**：
- 总 CBM（立方米）
- 总箱数
- 目的港
- 预计集货日期

**拼柜状态**：
```
POOL_PENDING(待拼箱) → POOL_READY(可安排) → CONTAINER_PLANNED(货柜计划已生成) → SHIPPED(已装柜)
```
