## 1. 需求背景

面向中日贸易场景的采购发注全链路管理系统，覆盖从「订单录入」到「日本签收/会计结算」的全流程数字化。

**核心价值链**（对应 Excel 设计）：

| Excel 表单 | 含义 |
|------------|------|
| 线下订单管理 | 1688 等外部订单的汇总入口 |
| 商品管理 | 商品目录（代码/尺寸/重量/包装/仓库） |
| 发注单主表 | 一次发注 = 多条商品行 + 状态追踪 |
| 货柜编号 | 集装箱管理 |
| 货物发送整理 | 装箱/发货整理 |

**价格计算公式**（前端实时计算，后端存结果）：
```
批发价 JPY = (人民币单价 ÷ 票点 × 1.02 × 1.2) × 汇率 × 1.05
```

## 2. 业务主表结构

### 2.1 发注单主表（Procurement）

```
一次发注 = 一个出货单（ShippingOrder）
  包含：下单日、厂家出货日、计划出货日
  包含：担当分配（日本担当/中国担当/商品担当）
  包含：多条商品明细行
  包含：状态追踪
```

### 2.2 商品目录表（Product）

```
商品代码（唯一键）
  名称（日文）
  高/宽/深（cm）
  尺寸总计 = 高 + 宽 + 深
  重量（kg）
  每包数量（段ボール入数）
  包装高/宽/深
  包装重量
  备注（如：箱规不固定 / 整托不固定）
  仓库（名古屋/久留米等）
  更新者 / 更新时间
```

## 3. 状态流转图

```
┌──────────────────────────────────────────────────────────────────┐
│                    永康路径（廠家直送）                              │
└──────────────────────────────────────────────────────────────────┘
                          →
    ┌──────────────────────────────────────────────────────────────┐
    │  発注待 → 永康 → 倉庫着 → 検品 → エア便 → 輸出 → 通関 → 日本着 → 会計  │
    └──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│                    OEM 路径                                         │
└──────────────────────────────────────────────────────────────────┘
                          →
    ┌──────────────────────────────────────────────────────────────┐
    │  発注待 → OEM → 倉庫着 → 現地検品 → メーカー直送 → 済      │
    └──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│                    状态说明                                         │
└──────────────────────────────────────────────────────────────────┘

  予定     — 还未下单，仅记录需求（未定 / OEM）
  発注待   — 已录入商品，等待下单
  直送     — 1688下单后，厂家直接发货（不经永康仓）
  OEM      — OEM 定制产品
  倉庫着   — 货物到达仓库
  現地検品 — 现场验货（异地）
  検品     — 仓库验货
  エア便   — 空运（尺寸/重量达标的轻量货）
  メーカー直送 — 厂家直送
  輸出     — 已出口
  通関     — 已报关
  日本着   — 已到日本
  会計     — 财务结算
  完了(済) — 全流程结束（终态）
  退货     — 退货管理
```

## 5. 状态推进规则

| 当前状态 | 触发动作 | 下一状态 |
|----------|----------|----------|
| 未定/予定 | 下单 | 発注待 |
| 未定/予定 | OEM下单 | OEM |
| 発注待 | 货物到达仓库 | 倉庫着 |
| 倉庫着 | 验货完成 | 検品 or 現地検品 |
| 現地検品 | 确认发货 | メーカー直送 |
| 検品 | 体积/重量判定 | エア便 / 輸出 |
| エア便 | 到达日本 | 通関 or 日本着 |
| 輸出 | 报关完成 | 通関 |
| 通関 | 清关完成 | 日本着 |
| 日本着 | 财务结算 | 会計 |
| 会計 | 结束 | 完了(済) |

## 6. 核心业务规则

| 规则 | 说明 |
|------|------|
| 状态终态 | 完了(済) 后不可修改 |
| 退货独立 | 退货与原单独立处理（Excel row 143） |
| 空运判定 | 尺寸+重量达标自动推荐（Excel row 91） |
| 永康路径 | 1688下单→永康→仓库→验货→发货 |
| 厂家直送 | 紧急/大批量不经仓库 |
| 1688集成 | 1688订单ID + 商品信息HP地址（Excel row 30/78） |
| OEM路径 | 新规→OEM→仓库→现场验货→直送→済 |
| 报价计算 | `(单价/票点×1.02×1.2)×汇率×1.05` |
