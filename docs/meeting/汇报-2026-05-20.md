## 第 2 张：项目亮点

### 亮点 1：DDD 六边形架构，领域边界清晰

```
interfaces/controller/   ← HTTP 适配器（Spring MVC）
application/usecase/     ← 用例（编排领域逻辑）
domain/model/           ← 实体、值对象、领域服务
domain/repository/      ← 仓储接口（Port）
infrastructure/persistence/jpa/  ← JPA 实现（Adapter）
```

- domain 层只引用 `libs/manpou-common`，禁止跨模块直接调用
- 违反架构约束 → ArchUnit 自动化检查

### 亮点 2：权限三角，前/后/数据库三层一致

| 层 | 机制 | 覆盖 |
|----|------|------|
| 前端 | `hasPermission('xxx:read')` 守卫 | 100% |
| 后端 | `@PreAuthorize` 注解 | 100% |
| 数据库 | `permission` 表 78 条记录 | 100% |

- 78 条权限，4 级角色（ADMIN/MANAGER/OPERATOR/VIEWER）
- JWT claim 透传，权限变更实时生效，无需重新登录

### 亮点 3：JWT RS256 跨服务认证

- user-service 签发 / allinone 验签，公钥 300s 轮询刷新
- 跨服务写审计日志：`X-AuditLog-Secret` 头校验
- 密钥来源统一：env var > classpath > DB

### 亮点 4：全量 i18n，中日双语

- 每一个前端标签都走 `$t()`，无硬编码
- zh.json / ja.json 双语完整，禁用直接显示枚举值

### 亮点 5：教训驱动开发（89 条 Lesson）

- 每踩一个坑 → 写一条 Lesson → 更新 `docs/lessons/README.md`
- 重启流程、枚举强转、SSL 拦截、JWT parse 等均有文档

### 亮点 6：十轮安全审计

- P0~P3 分级管理，共修复 27 个问题
- 从凭证泄露到 i18n 硬编码，完整覆盖

### 亮点 7：文档先于代码

- 每个功能都有 SPEC 文档（SPEC-B11/B12 等）
- UI 设计 + API 设计 + 数据库设计三合一

### 亮点 8：Phase 0 单体 → Phase 1 微服务平滑演进

- 按 Kafka Topic 边界拆分，不改业务逻辑，只改部署架构

## 第 8 张：遇到的问题与解决（根因分析）

**问题 1：allinone 启动失败（hibernate.dialect 缺失）**
- 根因：Spring Boot 3 无法自动推断 MySQL 方言
- 教训：框架版本特性不可假设，必须实测
- 解决：恢复 `hibernate.dialect: MySQLDialect`

**问题 2：COS 上传 500（Avast SSL 拦截）**
- 根因：Avast 企业版 SSL 中间人解密替换证书链
- 教训：新增第三方 HTTPS 调用必须 curl -v 先验证 TLS
- 解决：Avast 例外名单 / 临时关闭 / 导入根证书到 JDK

**问题 3：batch-categories ClassCastException**
- 根因：`Product.category` 是枚举，JPQL Object[] 强转 String 失败
- 教训：JPQL 返回枚举字段时结果是 Java 枚举对象
- 解决：`((ProductCategory) row[1]).name()`

**问题 4：JWT 跨服务验签 401**
- 根因：RS256 双重 parse bug + 公钥缓存失效
- 教训：JWT kid 提取必须从 header base64url 直接解码
- 解决：header base64url 直接解码 kid + 300s 轮询刷新
