## 2. 功能清单（按限界上下文）

### 2.1 发注上下文（核心）✅

**实体**：
- `PurchaseOrder`（采购单聚合根）：订单号/状态机/联系人/地址
- `ProcurementItem`（发注明细）：商品快照/数量/单价/材质/重量

**状态机**：
```
DRAFT → PENDING → APPROVED/REJECTED → SHIPPED → WAREHOUSE_PENDING
→ INSPECTION_OK → DISPATCHING → LOADED → CHINA_EXPORTED
```

**功能点**：
- [ ] 采购单新建/编辑/提交/审批
- [ ] 发注明细添加/删除/修改
- [ ] 新品准入标记（需手动录入数据库）
- [ ] 厂家信息联动（ProductService 事件监听）
- [ ] 分库管理（factoryCode 隔离）
- [ ] 采购单号自动生成（PO+yyyyMMdd+序号）

### 2.2 仓储上下文 ✅

**实体**：
- `ArrivalRecord`（到货记录聚合根）：到货单号/仓库/验收状态
- `CargoPhoto`（货物照片值对象）：照片类型/云端状态

**功能点**：
- [ ] 到货登记（永康=Yongkang）
- [ ] 验收人员指派（绿色标注）
- [ ] 验收开始/完成
- [ ] 照片上传云端（OSS/S3）
- [ ] 红色预警（priority=URGENT）

### 2.3 报关上下文 ✅

**实体**：
- `ExportDocument`（出口单据聚合根）：单据类型/状态/文件URL
- `Invoice`（商业发票值对象）：买卖双方/金额/条款
- `PackingList`（装箱单值对象）：货柜信息/重量/箱数

**功能点**：
- [ ] INVOICE 生成（含产品名/货号/数量/箱数/单价/材质/重量）
- [ ] PACKING LIST 生成
- [ ] 新品/换厂需高清图（attachedSheetRequired）
- [ ] 截单时效提醒

### 2.4 物流上下文 ✅

**实体**：
- `Container`（货柜聚合根）：柜号/类型/港口/装柜状态
- `ConsolidationPlan`（拼柜计划）：目的港/体积/厂家确认

**功能点**：
- [ ] 多口岸分流（青岛/宁波/厦门/深圳/上海）
- [ ] 拼柜计划制定
- [ ] 厂家确认（Ok3）后装柜
- [ ] 货柜封柜/开船/到港跟踪

### 2.5 财务上下文 ✅

**实体**：
- `TaxRefund`（退税单聚合根）：退税路径/金额/税率/发票状态
- `FeeDetail`（费用明细值对象）：费用类型/金额/认领状态

**功能点**：
- [ ] 退税申请（超慧/浙鲁/厂家/杂货）
- [ ] 退税票取得管理
- [ ] 未取得退税票高亮（isHighRisk()）
- [ ] 费用认领（海运费/检查费等）
- [ ] 凭证关联

### 2.6 通知上下文 ✅

**功能点**：
- [ ] 多渠道推送（短信/邮件/微信/站内消息）
- [ ] 通知模板管理
- [ ] 通知历史记录

## 3. 用户交互流程

### 3.1 采购单生命周期

```
用户录入采购单 → 新品判断 → 提交审核 → 审批通过/拒绝
→ 已发货至永康 → 到货登记 → 验收完成 → 调度
→ 装柜 → 开船 → 出口完成 → 退税处理
```

### 3.2 关键业务规则（需系统支撑）

| 规则 | 业务含义 | 实现要点 |
|------|---------|---------|
| 新品必填 | 新商品必须立即手动录入 | 采购单提交时校验 |
| 信息联动 | 厂家产品信息变动同步 | ProductService 事件监听 |
| 分库管理 | 单商品对多厂家时隔离 | factoryCode 隔离 |
| 红色预警 | 优先处理货物显示红色 | priority=URGENT |
| 绿色指派 | 验收人员绿色标注 | inspectorId 必填 |
| 照片存档 | 所有到货需拍摄照片 | photoUrls 必填 |
| INVOICE 必填项 | 出口单据合规 | Invoice VO 校验 |
| 截单时效 | 有新品时提前沟通 | 状态机+提醒 |
