# 角色分析：架构师（Architect）

> **视角**：技术决策 → 架构约束 → 可演进性 → 团队协作
> **核心问题**：系统能否支撑 20 年可维护性 + 100-500 人团队协作？

## 1. 架构总览

### 1.1 技术栈选型

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| 后端框架 | Spring Boot 3 + Spring Cloud Alibaba | 3.2.5 / 2023.0.1.2 | 微服务生态成熟 |
| ORM | Spring Data JPA + Hibernate | - | DDD 友好，审计字段自动化 |
| 数据库 | TiDB v8（生产）/ MySQL 8.x（开发） | 8.0 | 分布式 SQL |
| 数据库版本管理 | Flyway | 10.10.0 | SQL 脚本迁移 |
| 缓存/消息 | Redis + Kafka | 7 / 3.8 | 缓存 + 事件驱动 |
| 文件存储 | MinIO | latest | S3 兼容 |
| 链路追踪 | OTel Collector + Grafana Tempo | - | CNCF 标准 |
| 监控 | Prometheus + Grafana | - | 可视化 + 告警 |
| 网关 | Spring Cloud Gateway | - | 路由 + 鉴权 |
| 前端 | Vue 3 + TypeScript + Vite | 3.4 / 5.4 | Monorepo |
| Secret 管理 | K8s Secrets + ESO | - | 多后端支持 |

### 1.2 系统边界

```
[前端] → [API Gateway] → [Java 微服务群] → [MySQL / Redis / Kafka / MinIO]
                                        ↘ [Python 微服务群]（计算/AI）
                                                    ↓
                                           [Batch/Worker Service]
```

### 1.3 分层架构（每个微服务）

```
interfaces   → Controller（参数校验、调用 Application）
application  → UseCase / ApplicationService（编排业务）
domain       → Entity / Repository 接口（核心领域逻辑）
infrastructure → JPA 实现 / 配置 / 外部客户端 / Kafka
common       → 异常/日志/响应结构/工具
```

## 4. 关键架构约束

### 4.1 必须遵守的规范

| 约束 | 说明 | 强制方式 |
|------|------|---------|
| 审计字段 | 所有业务表必须包含 create_time/update_time/create_by/update_by/is_deleted | JPA Entity 基类 |
| 配置分离 | 所有配置走 Nacos，本地仅 .env.local | CI 检查 |
| TraceId 透传 | W3C traceparent 标准 | TraceFilter |
| 文件存储 | 大文件必须走 MinIO，禁止存数据库 | ArchUnit 规则 |
| ORM 模式 | JPA（生产环境 ddl-auto=validate） | 配置约束 |
| 统一响应 | 所有 API 返回 Result<T> | GlobalExceptionHandler |
| 幂等性 | POST + Kafka 消费者必须幂等 | @Idempotent 注解 |
| 架构验证 | CI 不通过 = 代码不可合并 | ArchUnit |

### 4.2 ArchUnit 架构规则

```java
// 核心约束
Application 只能被 Interfaces 访问
Application 只能访问 Common 和 Domain（禁止直接访问 Infrastructure）
Domain 只能被 Application 和 Infrastructure 访问
无循环依赖
```

### 4.3 分层依赖规则

```
Interfaces → Application → Domain ← Infrastructure
                ↓
            Common

禁止：Interfaces → Domain（必须经 Application）
禁止：Application → Infrastructure（通过 Port 接口解耦）
```

## 8. 可观测性架构

### 8.1 日志规范

- **格式**：结构化 JSON（logstash-logback-encoder）
- **必填字段**：time / level / service / traceId / thread / logger / msg
- **脱敏**：手机号/身份证/密码必须脱敏

### 8.2 链路追踪

- **标准**：W3C traceparent（traceparent: 00-{traceId}-{parentId}-{flags}）
- **实现**：micrometer-tracing-bridge-otel + OTel Collector
- **TraceId 注入**：TraceFilter（在 TraceId 为空时生成）

### 8.3 监控指标

| 指标类型 | 端点 | 说明 |
|---------|------|------|
| health | /actuator/health | 健康检查 |
| metrics | /actuator/metrics | 业务指标 |
| prometheus | /actuator/prometheus | Prometheus 采集 |
