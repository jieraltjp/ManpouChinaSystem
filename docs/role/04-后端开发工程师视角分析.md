## 3. 编码规范

### 3.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | UpperCamelCase | `PurchaseOrder` |
| 方法名 | lowerCamelCase | `findByOrderNo` |
| 常量 | UPPER_SNAKE_CASE | `ORDER_STATUS_DRAFT` |
| 包名 | lowercase | `com.manpou.procurement` |
| 枚举值 | UPPER_SNAKE_CASE | `DRAFT`, `PENDING` |

### 3.2 DTO 命名规范

| 类型 | 后缀 | 说明 |
|------|------|------|
| 创建命令 | `CreateCmd` | POST 请求体 |
| 更新命令 | `UpdateCmd` | PUT 请求体 |
| 查询条件 | `Query` | GET 查询参数 |
| 分页结果 | `PageQuery` | GET 返回体 |
| 登录请求 | `LoginCmd` | POST /auth/login |
| 登录响应 | `LoginVO` | 返回体 |

### 3.3 分层调用规范

```
Controller → UseCase → Repository(接口) → JPA实现
     ↓
  Assembler（DTO ↔ Entity）
```

**禁止**：
- Controller 直接调用 Repository
- UseCase 直接调用 JPA 实现类
- Domain 层依赖 Infrastructure 层

## 4. 核心代码模式

### 4.1 Entity 基类

```java
@MappedSuperclass
@EntityListeners(AuditingEntityListener.class)
public abstract class BaseEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "create_time", nullable = false, updatable = false)
    private LocalDateTime createTime;

    @Column(name = "update_time", nullable = false)
    private LocalDateTime updateTime;

    @Column(name = "create_by", length = 64, nullable = false)
    private String createBy = "";

    @Column(name = "update_by", length = 64, nullable = false)
    private String updateBy = "";

    @Column(name = "is_deleted", nullable = false)
    private Boolean isDeleted = false;
}
```

### 4.2 统一响应结构

```java
public class Result<T> {
    private String code;      // 错误码（字符串）
    private String message;
    private T data;
    private String traceId;
    private String detail;    // 详细错误信息

    public static <T> Result<T> success(T data) { ... }
    public static <T> Result<T> error(String code, String message) { ... }
}
```

### 4.3 幂等性注解

```java
@Idempotent(key = "#request.idempotentKey", expireSeconds = 86400)
public Result<OrderVO> createOrder(OrderCreateCmd cmd) { ... }
```

### 4.4 状态机（DDD）

```java
public class PurchaseOrder extends BaseEntity {
    private OrderStatus status;

    public void submit() {
        if (status != OrderStatus.DRAFT) {
            throw new BusinessException("ORDER_STATUS_ERROR", "只有草稿状态可提交");
        }
        this.status = OrderStatus.PENDING;
        // 发布领域事件
    }

    public void approve() {
        if (status != OrderStatus.PENDING) {
            throw new BusinessException("ORDER_STATUS_ERROR", "只有待审核状态可审批");
        }
        this.status = OrderStatus.APPROVED;
    }
}
```

### 6.1 测试分类

| 类型 | 命名 | 运行时机 |
|------|------|---------|
| 单元测试 | `*Test.java` | CI + 本地 |

### 6.2 测试覆盖要求

|------|-----------|------|

## 7. API 开发清单

### 7.1 新增一个 API 的标准流程

```
1. 创建 Flyway 迁移（V{n}__*.sql）
2. 创建 Entity（domain/model/）
3. 创建 Repository 接口（domain/repository/）
4. 创建 JPA 实现（infrastructure/persistence/）
5. 创建 DTO（application/dto/）
6. 创建 Assembler（application/assembler/）
7. 创建 UseCase（application/usecase/）
8. 创建 Controller（interfaces/controller/）
9. 添加单元测试
10. 运行 ArchUnit 测试
11. 更新 Swagger 文档
```

### 7.2 RESTful 规范

| 操作 | HTTP 方法 | 路径 | 说明 |
|------|---------|------|------|
| 列表 | GET | /api/v1/{resource} | 分页查询 |
| 详情 | GET | /api/v1/{resource}/{id} | 单条查询 |
| 新建 | POST | /api/v1/{resource} | 创建 |
| 更新 | PUT | /api/v1/{resource}/{id} | 全量更新 |
| 删除 | DELETE | /api/v1/{resource}/{id} | 软删除 |
| 自定义 | POST | /api/v1/{resource}/{id}/action | 业务动作 |
