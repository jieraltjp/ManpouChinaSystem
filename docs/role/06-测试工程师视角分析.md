## 1. 测试策略

### 1.1 测试金字塔

```
         ▲
        /E2E\        E2E 测试（少量，关键路径）
       /──────\
      /集成测试\    集成测试（API + DB）
     /──────────\
    / 单元测试   \   单元测试（大量，覆盖业务逻辑）
   /──────────────\
  / 架构测试(ArchUnit) \  ArchUnit（所有服务，CI 必须通过）
```

### 1.2 当前测试覆盖

| 类型 | 状态 | 说明 |
|------|------|------|
| ArchUnit 架构测试 | ✅ 8 个服务全部通过 | 分层约束 + 无循环依赖 |
| 单元测试 | 🔴 暂无业务测试 | 需覆盖 UseCase/Domain |
| 集成测试 | 🔴 暂无 | 需覆盖 Repository + DB |
| E2E 测试 | 🔴 暂无 | 需覆盖关键业务流程 |
| 契约测试 | 🔴 暂无 | 需覆盖 API 变更 |

### 4.2 API 测试示例（REST Assured）

```java
@Test
void shouldLoginSuccessfully() {
    given()
        .contentType(ContentType.JSON)
        .body("""
            {"username": "admin", "password": "admin123"}
            """)
    .when()
        .post("/api/v1/auth/login")
    .then()
        .statusCode(200)
        .body("data.accessToken", notNullValue())
        .body("data.tokenType", equalTo("Bearer"));
}

@Test
void shouldRejectInvalidCredentials() {
    given()
        .contentType(ContentType.JSON)
        .body("""
            {"username": "admin", "password": "wrong"}
            """)
    .when()
        .post("/api/v1/auth/login")
    .then()
        .statusCode(401);
}
```

### 4.3 状态机单元测试示例

> ⚠️ 以下为旧版测试示例（仅供参考）。v1.3.0 使用 `ShipmentStatus.canTransitionTo()` 校验。

```java
@Test
void shouldTransitionFrom未定Tö発注待() {
    // given
    Procurement p = new Procurement();
    p.setStatus(ShipmentStatus.未定);

    // when
    p.updateStatus(ShipmentStatus.発注待);

    // then
    assertThat(p.getStatus()).isEqualTo(ShipmentStatus.発注待);
}

@Test
void shouldRejectInvalidTransition() {
    // given
    Procurement p = new Procurement();
    p.setStatus(ShipmentStatus.未定);

    // when / then
    assertThatThrownBy(() -> p.updateStatus(ShipmentStatus.通了))
        .isInstanceOf(BusinessException.class)
        .hasMessageContaining("不允许跳转");
}

@Test
void shouldRejectModifyTerminalStatus() {
    // given
    Procurement p = new Procurement();
    p.setStatus(ShipmentStatus.完了);

    // when / then
    assertThatThrownBy(() -> p.updateStatus(ShipmentStatus.未定))
        .isInstanceOf(BusinessException.class)
        .hasMessageContaining("完了状态禁止");
}
```

## 5. 测试数据管理

### 5.1 测试数据策略

| 环境 | 数据来源 | 说明 |
|------|---------|------|
| 本地 H2 | Flyway 迁移 | 每次启动干净数据 |
| CI | @BeforeEach 初始化 | 每个测试独立数据 |
| 测试服务器 | SQL 脚本 | 可重复使用 |

### 5.2 测试隔离原则

- ✅ 每个测试独立：一个测试失败不影响其他测试
- ✅ 测试数据自包含：测试自己创建/清理数据
- ✅ 不依赖外部状态：Mock 所有外部依赖
- ❌ 禁止：测试间共享数据
- ❌ 禁止：测试依赖执行顺序

| 等级 | 定义 | 修复时限 | 示例 |
|------|------|---------|------|
| P0-Critical | 系统崩溃/数据丢失 | 1 小时 | 提交订单后数据库无记录 |

| P2-Medium | 功能有缺陷但可绕过 | 1 周 | 状态显示错误 |
| P3-Low | 界面/体验问题 | 1 个月 | 按钮样式偏差 |
