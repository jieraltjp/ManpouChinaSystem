# Step 3: 后端服务生成

> 本文档说明如何使用 Arch codegen 生成后端微服务

---

## 3.1 准备 codegen

> **注意**：`scaffolds/` 目录已从本项目删除（api-gateway 脚手架已迁移至 `apps/api-gateway/`）。
> codegen 工具位于外部 arch 仓库。ManpouChinaSystem 的 8 个微服务
> 均为预生成（已存在于 `apps/` 目录下），如需新增服务请从 arch 克隆。

### 3.1.1 安装 uv

```powershell
# Windows
pip install uv

# 或使用 winget
winget install --id=astral-sh.uv -e
```

### 3.1.2 获取 codegen

```powershell
# 方式一：从 arch 仓库克隆（推荐）
cd D:\Programme\java\ManpouChinaSystem
git clone https://your-git-server/arch-scaffolds.git scaffolds

# 或方式二：本地 arch 仓库（若已存在）
# cp -r D:/Programme/java/arch/scaffolds/codegen scaffolds/

# 安装依赖（克隆后进入 scaffolds/codegen 目录）
cd scaffolds\codegen
uv sync

# 验证
uv run codegen --help
```

**预期输出**：
```
Usage: codegen [OPTIONS] COMMAND [ARGS]...

  Arch 代码生成器 — 生成符合架构规范的微服务和模块。

  文档：https://github.com/company/arch

Options:
  --version  Show the version and exit.
  --help     Show this message and exit.

Commands:
  new-event    生成 Kafka 事件（Event 类 + Producer + Consumer）。
  new-module   在现有服务中生成领域模块（Entity + Repository + UseCase + Controller）。
  new-service  生成新的微服务脚手架。
```

## 3.2 生成用户服务

### 3.2.1 执行生成

```powershell
cd D:\Programme\java\ManpouChinaSystem

uv run codegen new-service `
    --name user-service `
    --lang java `
    --package com.manpou.user `
    --output ./apps
```

## 3.5 生成领域模块

### 3.5.1 用户模块

```powershell
# （在 arch 克隆的 scaffolds/codegen 目录下执行）
uv run codegen new-module `
    --name User `
    --service ..\..\apps\user-service `
    --fields "username:str,password:str,email:str,phone:str,status:int" `
    --description "用户实体"
```

**生成文件**：
```
apps/user-service/src/main/java/com/manpou/user/
├── domain/model/
│   ├── User.java                 # 用户实体
│   └── UserStatus.java          # 用户状态枚举
├── domain/repository/
│   └── UserRepository.java       # 用户仓储接口
├── application/dto/
│   ├── UserCreateCmd.java       # 创建命令
│   ├── UserUpdateCmd.java       # 更新命令
│   └── UserQuery.java           # 查询对象
├── application/assembler/
│   └── UserAssembler.java       # DTO ↔ Entity 转换
├── application/usecase/
│   └── UserUseCase.java         # 用户用例服务
├── interfaces/controller/
│   └── UserController.java     # REST 控制器
└── resources/db/migration/
    └── V2__add_user_table.sql   # Flyway 迁移
```

## 3.6 生成 Kafka 事件

### 3.6.1 订单创建事件

```powershell
uv run codegen new-event `
    --name OrderCreated `
    --producer order-service `
    --consumers inventory-service,message-service `
    --package com.manpou.order
```

**生成内容**：

**生产者端** (`order-service`)：
```
src/main/java/com/manpou/order/
├── domain/event/
│   └── OrderCreated.java           # 事件类
└── application/event/
    └── OrderCreatedProducer.java   # 事件发布器
```

**消费者端** (`message-service`)：
```
src/main/java/com/manpou/message/
└── application/event/
    └── OrderOrderCreatedConsumer.java  # 事件消费者
```

### 3.6.2 支付完成事件

```powershell
uv run codegen new-event `
    --name PaymentCompleted `
    --producer payment-service `
    --consumers order-service,message-service `
    --package com.manpou.payment
```

## 3.7 编译验证

### 3.7.1 编译用户服务

```powershell
cd D:\Programme\java\ManpouChinaSystem\apps\user-service

$env:JAVA_HOME = "C:\Program Files\Eclipse Adoptium\jdk-21.0.10+7-hotspot"
$env:MAVEN_HOME = "D:\Environment\apache-maven-3.9.15"
$env:Path = "$env:JAVA_HOME\bin;$env:MAVEN_HOME\bin;$env:Path"

# 清理编译
mvn clean compile

# 验证编译产物
Test-Path "target\classes\com\manpou\user\UserServiceApplication.class"
```

### 3.7.2 编译所有服务

```powershell
$services = @("user-service", "product-service", "order-service")

foreach ($svc in $services) {
    Write-Host "Building $svc..."
    cd "D:\Programme\java\ManpouChinaSystem\apps\$svc"
    mvn clean compile

    if ($LASTEXITCODE -eq 0) {
        Write-Host "  ✓ $svc compiled successfully" -ForegroundColor Green
    } else {
        Write-Host "  ✗ $svc build failed" -ForegroundColor Red
    }
}
```

### 3.7.3 运行测试

```powershell
cd D:\Programme\java\ManpouChinaSystem\apps\user-service

mvn test -Pskip-checkstyle
```
