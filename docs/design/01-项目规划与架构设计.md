## 1.1 项目概述

### 1.1.1 项目背景

**MANPOU** 是一套跨境贸易发注管理系统，包含：

| 模块 | 说明 |
|------|------|
| **发注管理** | 订单启动、新品准入、基础数据库维护 |
| **仓储验收** | 永康待验收、货物调度、人员指派 |
| **报关通关** | 出口单据生成、INVOICE/PACKING LIST |
| **物流调度** | 港口分流、拼柜管理 |
| **财务结算** | 退税管理、费用核算 |

### 1.1.2 团队角色

| 角色 | 姓名 | 职责 |
|------|------|------|
| 报关负责人 | 殷元 | 统筹通关全流程，材料合规、时效达标 |
| 辅料与库存协同 | 徐义超 | 辅料供应、库存可用性 |
| 退税与货柜管理 | 于世荣 | 退税优化、货柜周转效率 |
| 方案牵头人 | 张云 | 推动体系升级落地 |
| 财务与系统对接 | 许文豪 | 打通财务凭证流与系统数据流 |
| 系统协同支持 | 陈天仪 | 衔接业务需求与技术实现 |

### 1.1.3 技术选型

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| **后端框架** | Spring Boot 3 | 3.2.5 | Java 21 运行时 |
| **服务发现** | Nacos | 2.3 | 配置中心 + 注册中心 |
| **数据库** | MySQL 8 / TiDB | 8.0 | 关系型数据存储 |
| **缓存** | Redis | 7 | 会话缓存 + 分布式锁 |
| **消息队列** | Kafka | 3.8 | 事件驱动架构 |
| **前端** | Vue 3 + TypeScript | 3.4 | Composition API |
| **API 网关** | Spring Cloud Gateway | - | 统一入口 |
| **链路追踪** | OpenTelemetry + Tempo | - | 全链路可观测 |
| **文件存储** | OSS/S3 | - | 照片、单据云端存储 |

### 1.1.4 非功能性需求

| 指标 | 目标值 |
|------|--------|
| 可用性 | 99.9% |
| 响应时间 | P99 < 200ms |
| 并发用户 | 100+ |
| 数据量 | 支持百万级发注单 |

## 1.6 业务规则与校验

### 1.6.1 发注规则

| 规则 | 说明 | 实现 |
|------|------|------|
| 新品必填 | 新商品必须立即手动录入数据库 | 采购单提交时校验 |
| 信息联动 | 厂家产品信息变动需同步修改 | ProductService 事件监听 |
| 分库管理 | 单商品对多厂家时分库 | factoryCode 隔离 |
| 谈妥即下单 | 新品/换厂: "谈妥即下单、下单即开单" | 状态自动推进 |

### 1.6.2 验收规则

| 规则 | 说明 | 实现 |
|------|------|------|
| 红色预警 | 优先处理的货物显示红色 | priority = URGENT |
| 绿色指派 | 验收人员绿色标注 | inspectorId 必填 |
| 照片存档 | 所有到货需拍摄照片入群存档 | photoUrls 必填 |
| 空运推荐 | 体积小、重量轻的推荐空运 | weight < 5kg 自动提示 |

### 1.6.3 报关规则（殷元要求）

| 规则 | 说明 | 实现 |
|------|------|------|
| INVOICE内容 | 必须含: 产品名、货号、数量、箱数、单价、材质、重量 | Invoice VO校验 |
| ATTACHED SHEET | 新品/换厂/无关系配件必传高清图 | attachedSheetRequired |
| 截单时效 | 无新品: 截单前1小时; 有新品: 提前沟通 | 状态机+提醒 |

### 1.6.4 物流规则

| 规则 | 说明 | 实现 |
|------|------|------|
| 多口岸分流 | 青岛/宁波/厦门/深圳/上海 | Port 枚举 |
| 拼柜确认 | 验收完成+厂家确认(Ok3)后装柜 | FactoryConfirmation |
| 费用核算 | 海运费+检查费等4-5项 | FeeDetail 计算 |

### 1.6.5 财务规则（于世荣要求）

| 规则 | 说明 | 实现 |
|------|------|------|
| 退税分类 | 超慧/浙鲁/厂家/杂货 | RefundPath 枚举 |
| 高亮标注 | 未取得退税票订单高亮 | isHighRisk() |
| 凭证关联 | 退税票、原产地证关联至订单 | PaymentVoucher |
