## 2. 启动顺序（推荐）

### 2.1 依赖关系图

```
无硬依赖（各服务独立启动）
  │
  ├── user-service（18081）─────── 前端登录依赖
  │     （JWT 认证服务）
  │
  ├── procurement-service（18083）─── 发注管理核心
  │     （当前主要开发服务）
  │
  ├── product-service（18082）
  ├── warehouse-service（18084）
  ├── customs-service（18085）
  ├── logistics-service（18086）
  ├── finance-service（18087）
  └── notification-service（18088）
```

### 2.2 推荐启动顺序

```
第 1 批：基础设施（可选，需要时启动）
  redis（6379）          → 所有服务的会话/缓存
  mysql（3306）           → 所有服务的持久化数据（生产环境）

第 2 批：核心服务
  user-service（18081）   → 必须先启动，前端登录依赖
  procurement-service（18083） → 发注管理，登录后即可使用

第 3 批：配套服务（按需）
  product-service（18082）
  warehouse-service（18084）
  customs-service（18085）
  logistics-service（18086）
  finance-service（18087）
  notification-service（18088）

第 4 批：前端
  web（3000）             → 代理到 user-service（18081）
```

## 5. 快速验证

### 5.1 登录验证

```bash
# 直接调用 user-service
curl -X POST http://localhost:18081/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin"}'

# 通过前端代理
curl -X POST http://localhost:3000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin"}'
```

预期：返回包含 `accessToken` 的 JSON，code 为 `ok`。

### 5.2 前端访问

- 打开：http://localhost:3000
- 输入任意用户名/密码登录
- 侧边栏 →「采购单管理」查看测试页面
