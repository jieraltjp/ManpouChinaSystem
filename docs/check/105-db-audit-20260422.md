## 3. 问题分析

### P1: Flyway 全局禁用，V4 迁移不执行

**影响**: `V4__factory_migration.sql` 包含 500 条真实工厂数据，本应用于 MySQL 初始化，但 Flyway 被禁用。

**当前实际行为**:
- MySQL 生产环境：Hibernate `ddl-auto: update` 创建 `factory` 表（仅基础列），500条工厂数据不会导入
- H2 开发环境：Hibernate 创建 `factory` 表，V4 不会执行（MySQL 语法 H2 不兼容）

**结论**: 500条工厂数据当前**不会自动导入**任何环境。需要手动执行 V4 SQL 或启用 Flyway。

### P2: V1 示例表与实际 JPA 实体不一致

**V1 创建**: `example`, `product_example`, `warehouse_example`, `customs_example`, `logistics_example`, `finance_example`, `notification_example` — 骨架表，无业务价值。

**实际业务表**: 由 Hibernate `ddl-auto: update` 从 JPA 实体生成（`procurement`, `factory`, `qc_record`, `logistics_plan`, `replenishment_demand` 等）。

✅ V1 不影响实际业务（骨架表 + Hibernate 管理实际表）。

### P3: application.yml 注释误导

`application.yml` 第32-33行注释：
```yaml
# Flyway 已禁用：使用 Hibernate ddl-auto: update 管理表结构
# 如果需要迁移控制，修复 checksum 后将 enabled 改为 true
```
注释提到"修复 checksum"，暗示 Flyway 曾启用但被禁用。实际上 Flyway 从未在 manpou-allinone 中启用过。
