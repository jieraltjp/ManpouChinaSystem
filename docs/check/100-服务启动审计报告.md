## 2. 启动脚本

| 脚本 | 路径 | 职责 |
|------|------|------|
| `start-all.bat` | `scripts/` | 启动服务（支持单项 / 全部） |
| `restart-all.bat` | `scripts/` | 停止后重新启动（调用 stop + start） |

| `build-all.bat` | `scripts/` | 构建所有服务的 jar / node_modules |

### 启动命令

```bash
call scripts\start-all.bat all      # 启动全部三服务（默认）
call scripts\start-all.bat manpou   # 仅启动 manpou-allinone
call scripts\start-all.bat user     # 仅启动 user-service
call scripts\start-all.bat web      # 仅启动 web 前端
call scripts\start-all.bat status    # 仅查询状态
```

## 4. 启动流程（do_all 逻辑）

```
do_all
├── do_manpou  （端口 18090）
│   ├── 检查端口占用 → kill 占用进程
│   ├── 查找 target/manpou-allinone-*.jar
│   ├── jar 不存在 → mvn package -DskipTests
│   └── start /MIN java -jar ... --server.port=18090
├── do_user    （端口 18081）
│   ├── 检查端口占用 → kill 占用进程
│   ├── 查找 target/user-service-*.jar
│   ├── jar 不存在 → mvnw.cmd package / mvn package
│   └── start /MIN java -jar ... --server.port=18081
├── do_web     （端口 13000）
│   ├── 检查端口占用 → kill 占用进程
│   ├── node_modules 不存在 → npm install
│   └── start /MIN npm run dev
└── do_status  → 报告各服务运行状态
```

## 5. 健康检查

启动后通过 PowerShell 查询端口占用状态：

```powershell
Get-NetTCPConnection -LocalPort <PORT> -ErrorAction SilentlyContinue
```

超时阈值：30 秒轮询，超时则跳过等待。
