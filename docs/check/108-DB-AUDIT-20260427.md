## 2. 逐表详细问题

### 2.1 `replenishment_demand` — V31 未执行

**当前字段顺序（乱序）**:
```
id → demand_code → demand_type → product_code →
sub_product_code(TEXT, 应为VARCHAR) →
japan_lead → status → remarks →
create_time → update_time → create_by → update_by → is_deleted →
linked_demand_items(TEXT, 应删除) →
destination → linked_procurement_id → quantity
```

```
sub_product_code(VARCHAR) → quantity → destination →
japan_lead → status → linked_procurement_id → remarks →
create_time → update_time → create_by → update_by → is_deleted
```

### Phase 2 — 🟡 P1：类型修复

```sql
-- replenishment_demand: V31 迁移（TEXT → VARCHAR，删除 linked_demand_items）
-- Step 1: 加新列（临时，允许 NULL）
ALTER TABLE replenishment_demand
    ADD COLUMN sub_product_code_new VARCHAR(64) DEFAULT NULL COMMENT 'v2.0.0',
    ADD COLUMN quantity_new INT DEFAULT NULL COMMENT 'v2.0.0',
    ADD COLUMN destination_new VARCHAR(128) DEFAULT NULL COMMENT 'v2.0.0',
    ADD COLUMN linked_procurement_id_new BIGINT DEFAULT NULL COMMENT 'v2.0.0';

-- Step 2: 迁移数据（从旧 TEXT 列解析）
-- 2a. 单条子货号（如 "be"）→ sub_product_code_new
UPDATE replenishment_demand
SET sub_product_code_new = CONCAT(product_code, '-', TRIM(BOTH '"' FROM sub_product_code))
WHERE sub_product_code IS NOT NULL
  AND sub_product_code != ''
  AND sub_product_code NOT LIKE '[%'
  AND sub_product_code_new IS NULL;

-- 2b. JSON 数组中提取 quantity/destination
UPDATE replenishment_demand d,
(SELECT id,
        TRIM(BOTH '"' FROM SUBSTRING_INDEX(SUBSTRING_INDEX(d.sub_product_code, '"quantity":', -1), ',', 1)) AS qty,
        TRIM(BOTH '"' FROM SUBSTRING_INDEX(SUBSTRING_INDEX(d.sub_product_code, '"destination":', -1), '}', 1)) AS dest
 FROM replenishment_demand
 WHERE sub_product_code IS NOT NULL AND sub_product_code != '' AND sub_product_code LIKE '%"quantity":%') AS p
SET d.quantity_new = CAST(p.qty AS SIGNED),
    d.destination_new = p.dest,
    d.sub_product_code_new = CONCAT(d.product_code, '-',
        TRIM(BOTH '"' FROM SUBSTRING_INDEX(SUBSTRING_INDEX(d.sub_product_code, '"subCode":', -1), ',', 1)))
WHERE d.id = p.id AND d.quantity_new IS NULL;

-- 2c. 从 linked_demand_items JSON 提取 linked_procurement_id
UPDATE replenishment_demand d,
(SELECT id,
        CAST(TRIM(BOTH '"' FROM SUBSTRING_INDEX(
            SUBSTRING_INDEX(d.linked_demand_items, '"linkedProcurementId":', -1),
            ',', 1)) AS SIGNED) AS proc_id
 FROM replenishment_demand
 WHERE linked_demand_items IS NOT NULL AND linked_demand_items != ''
   AND linked_demand_items LIKE '%"linkedProcurementId":%') AS p
SET d.linked_procurement_id_new = p.proc_id
WHERE d.id = p.id AND d.linked_procurement_id_new IS NULL;

-- Step 3: 删除旧 TEXT 列
ALTER TABLE replenishment_demand
    DROP COLUMN IF EXISTS sub_product_code,
    DROP COLUMN IF EXISTS linked_demand_items;

-- Step 4: 重命名新列
ALTER TABLE replenishment_demand
    CHANGE COLUMN sub_product_code_new sub_product_code VARCHAR(64) DEFAULT NULL,
    CHANGE COLUMN quantity_new quantity INT DEFAULT NULL,
    CHANGE COLUMN destination_new destination VARCHAR(128) DEFAULT NULL,
    CHANGE COLUMN linked_procurement_id_new linked_procurement_id BIGINT DEFAULT NULL;

-- Step 5: 为新列加索引
ALTER TABLE replenishment_demand
    ADD INDEX idx_demand_quantity (quantity),
    ADD INDEX idx_demand_destination (destination),
    ADD INDEX idx_demand_sub_product_code (sub_product_code);

-- domestic_customs_record: TINYINT(1) → BIT(1)
ALTER TABLE domestic_customs_record MODIFY is_deleted BIT(1) NOT NULL DEFAULT b'0';

-- japan_customs_record: TINYINT(1) → BIT(1)
ALTER TABLE japan_customs_record MODIFY is_deleted BIT(1) NOT NULL DEFAULT b'0';

-- tax_refund_record: TINYINT(1) → BIT(1)
ALTER TABLE tax_refund_record MODIFY is_deleted BIT(1) NOT NULL DEFAULT b'0';

-- sales_record: TINYINT(1) → BIT(1)
ALTER TABLE sales_record MODIFY is_deleted BIT(1) NOT NULL DEFAULT b'0';

-- tax_refund_record: 精度修复
ALTER TABLE tax_refund_record
```

### Phase 6 — ⚪ P2：字段顺序（需表重建）

> MySQL 不支持在线修改列顺序。字段顺序对功能无影响，仅影响可读性/可维护性。
> 有数据的表（product=4999, factory=505, product_factory=710）重建风险较高，建议后续单独处理。
> 空表（qc_record=0, logistics_plan=0, domestic_customs_record=0, japan_customs_record=0, tax_refund_record=0, sales_record=1）可直接重建。
