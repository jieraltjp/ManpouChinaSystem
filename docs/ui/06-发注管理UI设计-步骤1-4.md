# 发注管理 UI 设计 — 步骤 1~4

> **版本**: 1.0.0
> **更新**: 2026-04-21
> **依据**: `DOMAIN-发注管理领域模型.md` v1.3.0 + 业务流分析
> **前提**: 基于现有橙色商业主题（`05-UI改版方案-橙色商业主题.md`）

## 1. 路由架构

### 1.1 路由清单

| 路由 | 页面 | 实体 | 入口 |
|------|------|------|------|
| `/procurement/demand` | DemandPage.vue | ReplenishmentDemand | 菜单：补货管理 |
| `/procurement/order` | OrderPage.vue | Procurement | 菜单：发注单管理 |
| `/procurement/factory` | FactoryPage.vue | Factory | 菜单：工厂管理 |
| `/procurement/inspection` | InspectionPage.vue | QcRecord | 菜单：验货记录 |
| `/procurement/logistics` | LogisticsPage.vue | LogisticsPlan | 菜单：调配计划 |
| `/procurement/order/:id` | OrderDetailPage.vue | Procurement | 发注单详情抽屉 |
| `/procurement/inspection/:id` | InspectionDetailPage.vue | QcRecord | 验货详情抽屉 |

### 1.2 侧边栏菜单结构

```
MANPOU
──────
🔶 发注管理
  ├─ 01 补货需求     /procurement/demand
  ├─ 02 发注单      /procurement/order
  ├─ 03 工厂管理     /procurement/factory
  ├─ 04 验货记录     /procurement/inspection
  └─ 05 调配计划     /procurement/logistics
```

## 4. 组件规范

### 4.1 状态 Tag 组件（StatusTag.vue）

统一封装所有状态显示，禁止直接使用 `<el-tag>`。

```vue
<!-- 用法 -->
<StatusTag :status="demand.status" type="demand" />
<StatusTag :status="procurement.status" type="shipment" />
<StatusTag :status="qcRecord.result" type="qc" />
<StatusTag :status="logistics.status" type="logistics" />
```

**props**:
- `status: String` — 状态枚举值
- `type: 'demand' | 'shipment' | 'qc' | 'logistics'` — 状态类型

### 4.2 商品选择器（ProductSelector.vue）

主货号 + 子货号（颜色）联动选择。

```vue
<ProductSelector
  v-model:masterCode="form.masterCode"
  v-model:subCode="form.subCode"
  @selected="onProductSelected"
/>
```

行为：
1. 用户输入/搜索主货号 → 下拉联想列表
2. 选中主货号后，自动加载该主货号下所有颜色变体
3. 子货号下拉框激活，显示 `re(红) / wh(白) / bk(黑)` 等选项
4. 选择完成后，触发 `@selected` 事件，填充商品名称、材质等只读字段

### 4.3 工厂选择器（FactorySelector.vue）

```vue
<FactorySelector
  v-model="form.factoryId"
  @selected="onFactorySelected"
/>
```

行为：
1. 下拉选择已有工厂
2. 支持关键字搜索
3. 选择后填充工厂位置、联系人、电话等只读字段
4. 支持内联新增工厂（点击 + 新增工厂 → 弹出 FactoryPage 的弹窗）

### 4.4 价格计算展示（PriceCalculator.vue）

前端实时计算批发价，公式：

```
估算批发价 JPY = (单价 ÷ 票点 × 1.02 × 1.2) × 汇率 × 1.05
```

```vue
<PriceCalculator
  :price-rmb="form.priceRmb"
  :tax-point="form.taxPoint"
  :exchange-rate="form.exchangeRate"
  :readonly="false"
/>
```

展示格式：`¥12,500 JPY`（橙色大字 + 小标签：含跨境费/利润/汇率）

### 4.5 验货不良率警示（DefectRateAlert.vue）

```
不良率 ≤ 1%：绿色（正常）
不良率 1%~3%：橙色（注意）
不良率 > 3%：红色（超标，阻止提交）
```

## 5. 数据流设计

### 5.1 第一步 → 第二步 数据流向

```
DemandPage [新建需求]
    ↓ 填写: demandType / productCode / subCode / quantity / destination / japanLead
    ↓ [转采购] 按钮
Procurement → OrderPage
    ↓ 自动带入: productCode / subCode / quantity / destination / japanLead
    ↓ 待用户补充: factoryId / priceRmb / taxPoint / billingType / orderDate / plannedShipDate / chinaLead
```

### 5.2 第二步 → 第三步 数据流向

```
OrderPage [发注单列表] → 点击验货按钮
    ↓ 选择验货方式（仓库验货 / 现场异地验货）
InspectionPage [新规验货]
    ↓ 自动带入: sellerName / productCode / productName / orderDate / destination / quantity / material
    ↓ 待用户录入: inspectionCount / boxCount / boxDimensions / qcResult / passedCount / defectiveCount / ...
```

### 5.3 第二步 → 第四步 数据流向

```
OrderPage [发注单列表] → 点击调配按钮
    ↓ 选择调配类型（海运 / 空运 / 拼柜）
LogisticsPage [新增调配]
    ↓ 自动带入: factoryId / productCode / requiresQc
    ↓ 待用户补充: cargoSize / cargoWeight / planType / estimatedShipDate
```
