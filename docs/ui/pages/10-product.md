## 1. 页面定位

商品主数据管理页面。录入/编辑商品基本信息（货号、品名、规格、HS编码、价格等），并维护商品与工厂的关联关系。商品数据是整个采购链路的起点。

## 3. 表格列定义（v1.2.0）

> **更新**: 2026-04-24（v1.2.0：调整列顺序；移除了库存/箱号；新增含税单价/票点列）

| 列名 | 字段 | 说明 |
|------|------|------|
| 主货号 | `masterCode` | 商品主货号，唯一标识 |
| 子货号 | `subCode` | 颜色/规格变体（可为空） |
| 图片 | `imageUrl` | 商品图片 |
| JAN码 | `janCode` | 日本商品条码 |
| 中文名称 | `nameZh` | 中文品名 |
| 英文名称 | `nameEn` | 英文品名 |
| 日文名称 | `nameJa` | 日文品名 |
| 原产地 | `origin` | 原产国/原产地 |
| 分类 | `category` | OEM / 普货 / 厂家直销 |
| 含税单价 | `unitPriceRmb` | 人民币含税单价，显示 `¥` 前缀 |
| 票点 | `taxPoint` | 票点值（如 1.1） |
| 材质 | `material` | 材质 |
| 需质检 | `requiresQc` | 是否需要质检 |
| 关联工厂 | `factoryCount` | 关联工厂数量（家），显示在抽屉中查看详情 |
| 操作 | — | 详情 / 编辑 / 删除 |

> **前端表格布局**：列宽用 `min-width`，不写 `table-layout="fixed"`，操作列不写 `fixed="right"`。详见 [docs/ui/ARCHITECTURE.md §8](../ARCHITECTURE.md#8-element-plus-表格布局规范)。

## 4. 新规录入 / 编辑弹窗

### 4.1 触发

- 新规：点击 `[+ 新规录入]` 按钮
- 编辑：表格操作列点击 `[编辑]`

### 4.2 表单字段

| 字段 | 控件 | 必填 | 说明 |
|------|------|------|------|
| 主货号 | `el-input` | ✅ | 最大32字符，唯一键 |
| 子货号 | `el-input` | | 颜色/规格，最大64字符 |
| 品名（中） | `el-input` | ✅ | 最大255字符 |
| 品名（英） | `el-input` | | 最大255字符 |
| 品名（日） | `el-input` | | 最大128字符 |
| HS编码 | `el-input` | | 最大20字符 |
| 单价(RMB) | `el-input-number` | | 精度2位，最小0 |
| 税点 | `el-input-number` | | 默认1.1，精度4位，最小0 |
| 退税率 | `el-input-number` | | 默认0.1，精度4位，最小0最大1 |
| 长(cm) | `el-input-number` | | 精度2位 |
| 宽(cm) | `el-input-number` | | 精度2位 |
| 高(cm) | `el-input-number` | | 精度2位 |
| 净重(kg) | `el-input-number` | | 精度4位 |
| 毛重(kg) | `el-input-number` | | 精度4位 |
| 单位 | `el-input` | | 最大50字符 |
| 产地 | `el-input` | | 最大100字符 |
| 申报要素 | `el-input[type=textarea]` | | 2行 |
| 备注 | `el-input[type=textarea]` | | 2行，最大512字符 |

### 4.3 提交

- 新规：`POST /api/v1/products`
- 编辑：`PATCH /api/v1/products/{id}`

## 5. 详情抽屉（右侧滑出，600px）

### 5.1 触发

表格操作列点击 `[详情]`。

### 5.2 内容结构

分三个区：

**基本信息区**
- 主货号 / 子货号 / 品名（中/英/日）/ JAN码 / HS编码 / 分类

**规格信息区**
- 原产地 / 材质 / 单位
- 含税单价 / 票点
- 尺寸：长×宽×高 cm（三个值任一有值才显示，否则显示 `—`）
- 重量：净重 / 毛重 kg（两个值任一有值才显示，否则显示 `—`）

**关联工厂区**
- 显示商品关联的工厂列表（来自 `product_factory` 关联表）
- 每个工厂卡片显示：工厂名 / 工厂编码 / 地区 / 联系人 / MOQ / 交期 / 单价
- 首选工厂显示 `el-tag[type=warning]` 标签
- 合作状态显示 `el-tag`
- 无关联工厂时显示"暂无关联工厂"

### 5.3 尾部按钮

- `[关闭]` — 关闭抽屉
- `[编辑]` — 从抽屉进入编辑弹窗

## 6. 删除操作

- 操作列 `[删除]` → `ElMessageBox.confirm` 确认框
- 确认后 `DELETE /api/v1/products/{id}`
- 成功提示 + 刷新列表

## 7. API 集成

| 操作 | Method | Endpoint |
|------|--------|----------|
| 分页查询 | GET | `/api/v1/products?keyword=&masterCode=&hsCode=&hsCodeJp=&factoryName=&page=&pageSize=` |
| 详情 | GET | `/api/v1/products/{id}` |
| 关联工厂 | GET | `/api/v1/products/{id}/factories` |
| 按货号查询 | GET | `/api/v1/products/code/{masterCode}` |
| 主货号补全 | GET | `/api/v1/products/suggest/master-codes?keyword=` |
| 子货号补全 | GET | `/api/v1/products/suggest/sub-codes?masterCode=` |
| 创建 | POST | `/api/v1/products` |
| 更新 | PATCH | `/api/v1/products/{id}` |
| 删除 | DELETE | `/api/v1/products/{id}` |
