## 2.1 开发环境要求

- **Windows 11** (当前开发环境)

- **macOS** (可选)

| 工具 | 安装路径 |

| JDK | `C:\Program Files\Eclipse Adoptium\jdk-25.0.2.10-hotspot` |

| Node.js | `C:\nvm4w\nodejs` |
| Python | `C:\Users\<User>\AppData\Local\Programs\Python\Python313` |

### 2.2.2 配置环境变量

**PowerShell (推荐)**：
```powershell
# 创建系统环境变量（需要管理员权限）
[Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Eclipse Adoptium\jdk-25.0.2.10-hotspot", "Machine")
[Environment]::SetEnvironmentVariable("MAVEN_HOME", "D:\Environment\apache-maven-3.9.15", "Machine")
[Environment]::SetEnvironmentVariable("Path", "$env:Path;C:\Program Files\Eclipse Adoptium\jdk-25.0.2.10-hotspot\bin;D:\Environment\apache-maven-3.9.15\bin", "Machine")

# 刷新环境变量
$env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")
```

**或在 `~/.bashrc` / `~/.zshrc`**：
```bash
# Git Bash / WSL
export JAVA_HOME="/c/Program Files/Eclipse Adoptium/jdk-25.0.2.10-hotspot"
export MAVEN_HOME="/d/Environment/apache-maven-3.9.15"
export PATH="$JAVA_HOME/bin:$MAVEN_HOME/bin:$PATH"
```

## 2.3 项目目录初始化

### 2.3.1 创建项目根目录

```powershell
# 创建项目目录
New-Item -ItemType Directory -Path D:\Programme\java\ManpouChinaSystem -Force

# 创建子目录
$dirs = @(
    "apps",
    "config",
    "docker",
    "scripts",
    "docs",
    ".github\workflows"
)

foreach ($dir in $dirs) {
    New-Item -ItemType Directory -Path "D:\Programme\java\ManpouChinaSystem\$dir" -Force
}

# 查看目录结构
Get-ChildItem -Path D:\Programme\java\ManpouChinaSystem -Recurse
```

### 2.3.2 克隆脚手架仓库

```powershell
cd D:\Programme\java\ManpouChinaSystem

# 方式1: 克隆 Arch 脚手架仓库
git clone https://your-git-server/arch-scaffolds.git scaffolds

# 方式2: 复制现有脚手架
Copy-Item -Recurse -Path "D:\Programme\java\arch\scaffolds\*" -Destination "D:\Programme\java\ManpouChinaSystem\scaffolds\"
```

## 2.4 数据库准备

### 2.4.1 Docker 启动 MySQL

```powershell
docker run -d `
    --name manpou-mysql `
    -p 3306:3306 `
    -e MYSQL_ROOT_PASSWORD=root123 `
    -e MYSQL_DATABASE=manpou `
    -v manpou-mysql-data:/var/lib/mysql `
    mysql:8.0 `
    --default-authentication-plugin=mysql_native_password
```

### 2.4.2 初始化脚本

创建 `docker/init.sql`：
```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS manpou CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 使用数据库
USE manpou;

-- 创建用户表
CREATE TABLE IF NOT EXISTS `user` (
    `id` BIGINT NOT NULL AUTO_INCREMENT PRIMARY KEY,
    `username` VARCHAR(32) NOT NULL UNIQUE,
    `password` VARCHAR(128) NOT NULL,
    `email` VARCHAR(64) NOT NULL,
    `phone` VARCHAR(20),
    `status` TINYINT NOT NULL DEFAULT 0,
    `create_time` DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    `update_time` DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    `create_by` VARCHAR(64) NOT NULL DEFAULT '',
    `update_by` VARCHAR(64) NOT NULL DEFAULT '',
    `is_deleted` TINYINT NOT NULL DEFAULT 0,
    INDEX `idx_username` (`username`),
    INDEX `idx_email` (`email`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 2.7 常见问题

### Q1: Maven 找不到 `mvn` 命令

**解决**：
```powershell
# 确认 Maven 安装路径存在
Test-Path "D:\Environment\apache-maven-3.9.15\bin\mvn.cmd"

# 添加到 PATH
[Environment]::SetEnvironmentVariable(
    "Path",
    [System.Environment]::GetEnvironmentVariable("Path", "Machine") + ";D:\Environment\apache-maven-3.9.15\bin",
    "Machine"
)
```

### Q2: Java 版本过低

**解决**：
```powershell
# 检查 JAVA_HOME
$env:JAVA_HOME

# 设置正确的 JAVA_HOME
[Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Eclipse Adoptium\jdk-25.0.2.10-hotspot", "Machine")

# 重启终端或刷新环境变量
refreshenv
```

### Q3: Docker 启动失败

**解决**：
```powershell
# 以管理员身份运行 PowerShell
Start-Process powershell -Verb RunAs

# 重启 Docker Desktop 服务
Restart-Service com.docker.service
```
